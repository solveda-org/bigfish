01/17/2013 - Issue# 28076 -- Swap the Column ProductId and ProductIdTo values
/*** In ProductAssoc entity the PRODUCT_ID would have the Id of Main Product and the PRODUCT_ID_TO will contain the Id of Associated Products.
     For Ex. PRODUCT_ID    PRODUCT_ID_TO
             1001          1003
             1001          1004
     If the Values are not in this order then the below query will flip the Column PRODUCT_ID and PRODUCT_ID_TO values. ***/
      
    UPDATE IGNORE PRODUCT_ASSOC SET PRODUCT_ID=@tmp:=PRODUCT_ID, PRODUCT_ID=PRODUCT_ID_TO, PRODUCT_ID_TO=@tmp where PRODUCT_ASSOC_TYPE_ID = 'PRODUCT_COMPLEMENT';
    - NOTE:RB-1/31/2013 The 'IGNORE' in the sql above is needed for the case that there are two products that both have each other as complements. In this case, running this statement (without 'IGNORE') could cause a duplicate entry error.  The IGNORe keyword will do nothing when it runs into this error and continue to process the remaining rows.
